  Here's what each script now does:
  - check-network.js: Loads a network definition from a JSON file and displays its properties
  - basic-oscillator.js: Creates a simple 2-node oscillator network and runs a simulation
  - load-netdef.js: Loads and runs a network from a JSON definition file with command-line arguments
  - direct-oscillator.js: Creates an oscillator manually with backend selection
  - terminal-visualize.js: Visualizes a simulation with terminal-based sparklines
  - python-port-oscillator.js: A direct port of Python CTRNN code
  - term-sparklines.js: Utility for terminal-based network visualization
