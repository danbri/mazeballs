How It Works

This streaming demo uses an asynchronous generator pattern to continuously output simulation results in real-time. Unlike the basic example that collects all data before displaying, this demo:

  • Processes each simulation step as it happens
  • Updates the visualization in real-time
  • Monitors GPU memory usage to prevent memory leaks
  • Implements a sliding window approach to limit memory consumption
  • Can run indefinitely without crashing

The streaming approach is especially useful for long-running simulations or when monitoring a network's response to external inputs over time.
